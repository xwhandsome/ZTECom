# 夜间不适处理规则

**适用问题**：用户说“我今晚不舒服”“老人不舒服怎么办”“夜里觉得冷/热/黑/害怕”时，命中本卡片。

**处理原则**：系统先区分可由环境设备缓解的问题和需要家属介入的问题。太冷、太热可联动空调；太暗、起夜可联动灯光；无法通过环境设备处理的不适，应建议联系家属或医生。

**安全边界**：本系统不能判断疾病严重程度，不能替代医疗诊断。出现胸痛、呼吸困难、意识异常、严重跌倒、持续出血等急症信号时，应及时联系家属、医生或急救服务。

**关键词**：夜间不适，不舒服，太冷，太热，太暗，起夜，急症，联系家属。

# 通知家属二次确认规则

**适用问题**：用户要求“通知我儿子”“告诉家属”“联系女儿”时，命中本卡片。

**处理原则**：通知家属属于高影响工具调用，必须二次确认。系统应先复述联系人和通知内容，等待用户确认后再执行模拟通知；用户取消或确认编号不匹配时，不执行通知。

**演示口径**：演示版只做本地模拟通知，不发送真实短信、不拨打真实电话。工具事件会记录在本地 SQLite，便于页面展示和答辩追溯。

**关键词**：通知家属，联系儿子，联系女儿，二次确认，模拟通知，确认执行，取消。

# 传感器状态解释

**适用问题**：用户询问“卧室温度怎么样”“湿度多少”“活动状态正常吗”“传感器数据从哪里来”时，命中本卡片。

**数据说明**：v1 演示中的传感器数据由本地模拟状态提供，包括卧室、客厅的温度、湿度和活动状态。数据用于展示端侧 Agent 的场景感知和工具调用链路，不代表真实医疗监测设备数据。

**回答口径**：系统可以回答当前模拟温湿度和活动状态，也可以基于环境规则控制设备；不能把模拟传感器数据解释为医学诊断。

**关键词**：传感器，卧室温度，客厅温度，湿度，活动状态，模拟数据，场景感知。

# 本地照护隐私规则

**适用问题**：用户询问“数据会不会上传”“联系人保存在哪里”“对话历史在哪里”时，命中本卡片。

**隐私说明**：本项目的提醒、设备状态、联系人、环境规则、工具事件和会话上下文保存在本地 SQLite 中。Java 展示层只通过 localhost 调用 Python API，不接外部 AI 接口。

**安全边界**：演示版不发送真实短信，不拨打真实电话，不接真实外设，不提供真实医疗诊断。

**关键词**：隐私，本地 SQLite，离线，localhost，家庭数据，联系人，对话历史。
