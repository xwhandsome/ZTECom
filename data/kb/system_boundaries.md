# 端侧本地运行说明

**适用问题**：用户询问“系统是不是离线运行”“端侧是什么意思”“为什么不用云端”时，命中本卡片。

**系统口径**：本项目的核心链路在本地执行，包括规则 NLU、槽位补全、Agent 规划、工具调用、SQLite 记忆和本地知识库检索。Java 展示层只通过 localhost 调用 Python API，不承载模型逻辑。

**比赛卖点**：端侧运行可以降低隐私数据外传风险，减少云端依赖，并便于在低资源环境下展示稳定响应。

**关键词**：端侧，本地运行，离线，localhost，Python API，Java 展示层。

# 隐私保护说明

**适用问题**：用户询问“联系人会不会上传”“对话会不会出网”“家庭数据保存在哪里”时，命中本卡片。

**系统口径**：提醒、设备状态、传感器模拟数据、环境规则、工具调用和对话历史保存在本地 SQLite。演示版不上传家庭隐私数据，不接云端 LLM，不发送真实短信或电话。

**边界说明**：如果后续接入真实短信、电话、硬件或云端模型，需要重新设计授权、日志脱敏和隐私说明。

**关键词**：隐私，本地 SQLite，联系人，对话历史，不出网，家庭数据。

# 本地知识库说明

**适用问题**：用户询问“RAG 是怎么做的”“知识来源是什么”“为什么回答里有引用”时，命中本卡片。

**系统口径**：本项目采用轻量知识卡片 RAG。知识卡片保存在 `data/kb`，系统按 Markdown 标题切片，通过关键词和上下文词检索，返回卡片标题、chunk_id 和摘要片段。

**边界说明**：当前不是向量数据库 RAG，不依赖 embedding 模型。优点是轻量、可解释、离线；缺点是对隐晦同义表达的泛化能力弱，需要人工维护高质量卡片。

**关键词**：RAG，知识卡片，本地知识库，引用，chunk_id，关键词检索。

# 模型降级策略

**适用问题**：用户询问“没有模型能不能跑”“LLM 失败怎么办”“为什么 LLM 未调用”时，命中本卡片。

**系统口径**：系统默认规则优先。只有规则无法完整解析时，才尝试本地 GGUF 模型输出严格 JSON。没有模型、模型路径错误、DLL 预加载失败或 JSON 解析失败时，系统自动降级到规则、工具和本地 RAG。

**展示说明**：健康接口会展示 LLM 是否启用、模型路径是否存在、runtime 状态和降级原因。

**关键词**：模型降级，GGUF，llama.cpp，规则优先，LLM 未调用，runtime。

# 真实医疗建议边界

**适用问题**：用户询问“系统能不能判断病情”“能不能给药物建议”“剂量是否合理”时，命中本卡片。

**系统口径**：本系统不能诊断疾病，不能开药，不能替代医生或药师。用药相关回答只能引用本地知识卡片，并提醒以医生医嘱、药品说明书或药师建议为准。

**风险处理**：出现急症信号、严重不良反应或高风险人群用药问题时，应建议联系医生、药师、家属或急救服务。

**关键词**：医疗边界，不能诊断，不能开药，医生医嘱，药师，说明书。

# 家属通知模拟边界

**适用问题**：用户询问“通知家属会不会真的发送”“为什么要确认”“通知记录在哪里”时，命中本卡片。

**系统口径**：演示版只做本地模拟通知。通知家属前必须二次确认，确认后写入本地工具事件，Java 页面可以展示最近通知工具调用。

**边界说明**：演示版不发送真实短信、不拨打真实电话。真实通知能力需要额外接入通信服务、权限确认和失败重试机制。

**关键词**：家属通知，模拟通知，二次确认，工具事件，短信，电话。
